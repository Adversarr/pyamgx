Quick start¶

Initializing and finalizing pyamgx¶

The initialize() and finalize() functions must be called to initialize and finalize the library respectively.

import pyamgx
pyamgx.initialize()

# use pyamgx

pyamgx.finalize()

Config objects¶

Config objects are used to store configurations for Resource and Solver objects. Config objects can be constructed from JSON files or dict objects. Example configurations can be found here.

cfg = pyamgx.Config()
cfg.create_from_file("/path/to/config.json")
cfg.create_from_dict({"max_levels": 10})

After use, Config objects must be destroyed using the destroy() method.

cfg.destroy()

Resources objects¶

Resources objects are used to specify the resources (GPUs, MPI ranks) used by Vector, Matrix and Solver objects. Currently, pyamgx only supports “simple” Resources objects for single threaded, single GPU applications. created using the create_simple() method:

resources = pyamgx.Resources()
resources.create_simple(cfg)

After use, Resources objects must be destroyed using the destroy() method after all Vector, Matrix and Solver classes constructed from the Resouces object are destroyed.

# destroy Solver, Matrix and Vector objects first.
resources.destroy()

Vectors¶

Vector objects store vectors on either the host (CPU memory) or device (GPU memory). The value of the mode argument to the create() method specifies whether the data resides on the host or device.

vec = pyamgx.Vector()
vec.create(resources, mode='dDDI')

Values of Vector objects can be populated in two ways:

  1. From NumPy numpy.ndarray using the upload() method.
  2. Using the set_zero() method.
